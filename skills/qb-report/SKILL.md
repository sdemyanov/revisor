# Revisor QuickBooks Report

Your job is to generate owner-facing and shareable Business Health Report
artifacts using only the QuickBooks surface and saved context that Revisor has
actually proven.

Use the shared rules from:

- `skills/02-cash-flow-forecasting/SKILL.md`
- `skills/03-financial-health/SKILL.md`
- `skills/11-proactive-analysis/SKILL.md`
- `skills/10-formatting/SKILL.md`
- `skills/12-health-scoring/SKILL.md`

Also follow the artifact rules in:

- `phase1/report-artifact-contract.md`

If those files are not available in the runtime, follow the condensed rules below directly.

## Guardrails

- Read-only QuickBooks only. Never call `quickbooks-profile-info-update` or `quickbooks-transaction-import`.
- Read `.revisor/context.json` if present.
- Read recent snapshots if present for trend arrows.
- Do not write or modify snapshots from `/revisor:qb-report`.
- Write four canonical report artifacts:
  - internal markdown
  - internal HTML
  - shareable markdown
  - shareable HTML
- Use only data the current connector can actually support.

## Required Inputs

1. Load `.revisor/context.json` if it exists.
2. Load recent snapshots if present for trend direction.
   - Treat `qb-setup`, `qb-snapshot`, and `qb-weekly` snapshots as comparable
     when the core metric set matches.
   - Same-day snapshots still count as prior comparable history when the metric
     set matches.
   - If at least one prior comparable snapshot shows the same state, use `→`
     rather than `?`.
   - If the latest comparable snapshot has identical or materially unchanged
     values, the comparable scored dimensions should normally show `→`.
3. Confirm these tools are available:
   - `company-info`
   - `profit-loss-quickbooks-account`
   - `cash-flow-quickbooks-account`
4. If any required tool is missing, stop with a clear unblock message and do not write partial artifacts.
5. Call `company-info`.
6. Pull a trailing 12 complete month P&L window:
   - if today is mid-month, end at the last day of the prior month
   - otherwise end today
7. Pull cash flow twice:
   - same trailing 12 complete month window for comparable operating picture
   - a fresh window ending today so `cashAtEnd` is as current as the connector allows

## QuickBooks-Only Scorecard

Score these 5 dimensions using `skills/12-health-scoring/SKILL.md`:

1. `Cash Runway`
2. `Revenue Activity`
3. `Margins`
4. `Expense Stability`
5. `Obligation Readiness`

Use `N/A` honestly when a dimension is not meaningful, for example zero-revenue margin scoring in a dormant entity.

Compute:

- each dimension grade
- each dimension arrow
- overall grade from scored dimensions only

Do not describe unchanged same-day comparable history as `insufficient` just
because it is same-day. Use `?` only when there is truly no comparable prior
snapshot.

## Artifact Writes

Write these files using one shared UTC timestamp:

- `.revisor/reports/<UTC timestamp>-health-report-internal.md`
- `.revisor/reports/<UTC timestamp>-health-report-internal.html`
- `.revisor/reports/<UTC timestamp>-health-report-shareable.md`
- `.revisor/reports/<UTC timestamp>-health-report-shareable.html`

### Internal Variants

Internal variants may include:

- exact dollar amounts
- exact runway values
- exact percentages
- account/category names
- concrete risks and recommendations

### Shareable Variants

Shareable variants must not include:

- dollar amounts
- cash balances
- revenue totals
- exact percentages when they reveal sensitive detail
- exact customer or vendor names
- exact account identifiers

Shareable variants may include:

- dimension names
- grades
- trend arrows
- qualitative explanations
- relative phrases like `above target`, `watch`, or `needs attention`
- one next-step recommendation
- footer CTA:
  - `Generated by Revisor`
  - `Search "Revisor" in Cowork`
  - `Open-source community project`

### HTML Rules

- write standalone HTML directly
- inline CSS only
- no external assets
- keep the design clean and screenshot-friendly

## Output Structure

Use `skills/11-proactive-analysis/SKILL.md` when choosing the strongest
`Top strengths`, `Top risks`, and the single recommended next step.

Both markdown variants should include:

1. Title and reporting date
2. Overall grade
3. Five dimension rows with grade and arrow
4. Top strengths
5. Top risks
6. One recommended next step
7. Footer

## Unsupported Areas

Do not fabricate grades for:

- collections
- concentration
- pipeline health

Those require data the current QuickBooks-only connector does not expose.

If useful, say that the current QuickBooks-only report is a 5-dimension scorecard based on the proven connector surface.

## Output To User

After writing the four files, reply briefly with:

1. `Revisor QuickBooks Report`
2. overall grade
3. the 5 dimension grades in one compact list
4. files written
5. one-sentence note about the main risk driving the report
