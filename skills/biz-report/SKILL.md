# Revisor Business Report

Your job is to generate owner-facing and shareable Business Health Report
artifacts using the currently connected Revisor sources without fabricating
unsupported dimensions.

Use the shared rules from:

- `skills/02-cash-flow-forecasting/SKILL.md`
- `skills/03-financial-health/SKILL.md`
- `skills/04-pipeline-analytics/SKILL.md`
- `skills/06-subscription-intelligence/SKILL.md`
- `skills/07-cross-source-synthesis/SKILL.md`
- `skills/11-proactive-analysis/SKILL.md`
- `skills/12-health-scoring/SKILL.md`
- `skills/13-biz-scoring/SKILL.md`
- `skills/10-formatting/SKILL.md`

Also follow the artifact rules in:

- `phase1/report-artifact-contract.md`

If those files are not available in the runtime, follow the condensed rules
below directly.

## Guardrails

- Read-only only. Never call mutating connector tools.
- Read `.revisor/context.json` if present.
- Read recent snapshots if present for trend arrows.
- Do not write or modify snapshots from `/revisor:biz-report`.
- Write four canonical report artifacts:
  - internal markdown
  - internal HTML
  - shareable markdown
  - shareable HTML
- Use only data the current connectors can actually support.

## Required Inputs

1. Load `.revisor/context.json` if it exists.
2. Detect connected sources from `source_accounts`.
3. Load recent comparable snapshots when available:
   - QuickBooks snapshots for the first five dimensions
   - HubSpot weekly snapshots for `Pipeline Health`
4. Use the freshest safe reads available from connected sources.

## Current Cross-Source Scorecard

Score these 6 dimensions using `skills/13-biz-scoring/SKILL.md`:

1. `Cash Runway`
2. `Revenue Activity`
3. `Margins`
4. `Expense Stability`
5. `Obligation Readiness`
6. `Pipeline Health`

Rules:

- the first 5 dimensions should upgrade cleanly from the current QuickBooks-only
  report
- `Pipeline Health` is the sixth dimension when HubSpot is available
- if HubSpot is unavailable, `Pipeline Health` should be `N/A`
- Stripe may influence strengths, risks, and the next step, but it does not add
  a separate scored dimension in this version
- compute overall grade from scored dimensions only

## Artifact Writes

Write these files using one shared UTC timestamp:

- `.revisor/reports/<UTC timestamp>-biz-health-report-internal.md`
- `.revisor/reports/<UTC timestamp>-biz-health-report-internal.html`
- `.revisor/reports/<UTC timestamp>-biz-health-report-shareable.md`
- `.revisor/reports/<UTC timestamp>-biz-health-report-shareable.html`

### Internal Variants

Internal variants may include:

- exact dollar amounts
- exact runway values
- exact percentages
- source names and product names
- concrete risks and recommendations

### Shareable Variants

Shareable variants must not include:

- dollar amounts
- cash balances
- revenue totals
- exact percentages when they reveal sensitive detail
- exact customer or vendor names
- exact account identifiers

Shareable variants may include:

- dimension names
- grades
- trend arrows
- qualitative explanations
- relative phrases like `watch`, `critical`, `stalled`, or `needs attention`
- one next-step recommendation
- footer CTA:
  - `Generated by Revisor`
  - `Search "Revisor" in Cowork`
  - `Open-source community project`

### HTML Rules

- write standalone HTML directly
- inline CSS only
- no external assets
- keep the design clean and screenshot-friendly

## Output Structure

Use `skills/11-proactive-analysis/SKILL.md` when choosing the strongest
`Top strengths`, `Top risks`, and the single recommended next step.

Both markdown variants should include:

1. Title and reporting date
2. Sources used
3. Overall grade
4. Six dimension rows with grade and arrow
5. Top strengths
6. Top risks
7. One recommended next step
8. Footer

## Output To User

After writing the four files, reply briefly with:

1. `Revisor Business Report`
2. overall grade
3. the 6 dimension grades in one compact list
4. files written
5. one-sentence note about the main cross-source risk driving the report
